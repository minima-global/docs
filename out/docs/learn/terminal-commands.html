Terminal Commands

Last updated: 6th August 2024

The following commands can be run directly from a Minima Terminal. They can also be viewed by typing help into the Minima Terminal.

For detailed help, use help command: followed by the command.

; can be used to chain multiple commands

General Commands

Balance

Show your total balance of Minima and tokens.

ParametersDescription
address
(Optional) Show the balance for a specific 0x or Mx address.
tokenid
(Optional) Show the balance for a specific tokenid. Minima is 0x00.
confirmations
(Optional) Set the number of block confirmations required before a coin is considered confirmed in your balance. Default is 3.

Examples:

Block

Return the top block

Checkaddress

Check the validity of an address and whether it belongs to your node.

ParametersDescription
address
The 0x or Mx address to check.

Examples:

Coincheck

Check a coin exists and is valid. Can only check unspent coins.

ParametersDescription
data
The data of a coin. Can be found using the `coinexport` command.

Examples:

Coinexport

Export a coin including its MMR proof.

ParametersDescription
coinid
The id of a coin. Can be found using the `coins` command.

Examples:

Coinimport

Import a coin including its MMR proof.

ParametersDescription
data
The data of a coin. Can be found using the `coinexport` command.
track
(Optional) true or false, true will create an MMR entry for the coin and add it to your relevant coins. Default is false.

Examples:

Cointrack

Track or untrack a coin.

ParametersDescription
enable
true or false, true will add the coin to your relevant coins, false will remove it from your relevant coins.
coinid
The id of a coin. Can be found using the `coins` command.

Examples:

Consolidate

Consolidate multiple coins (UTxOs) into one by sending them back to yourself.

Must have at least 3 coins.

Useful to prevent having many coins of tiny value and to manage the number of coins you are tracking. Optionally set the minimum coin age (in blocks), maximum number of coins, and maximum number of signatures for the transaction.

ParametersDescription
tokenid
The tokenid for Minima or custom token to consolidate coins for. Minima is 0x00.
coinage
(Optional) The minimum number of blocks deep (confirmations) a coin needs to be. Default is 3.
maxcoins
(Optional) The maximum number of coins to consolidate. Minimum 3, up to 20. Coins are first sorted by value (smallest first) before adding to the transaction. Default is 20.
maxsigs
(Optional) The maximum number of signatures for the transaction, up to 5. Coins are then sorted by address to minimize the number of signatures required. Default is 5.
burn
(Optional) Amount of Minima to burn with the transaction. Default is 0.
debug
(Optional) true or false, true will print more detailed logs. Default is false.
dryrun
(Optional) true or false, true will simulate the consolidate transaction but not execute it. Default is false.

Examples:

Getaddress

Returns an existing default Minima address to receive funds, use as a change address etc.

Each address can be used securely 262144 (64^3) times.

Then you can wipe the private keys from your online node using the 'vault' command.

Examples:

Hashtest

Check the speed of hashing of this device.

Defaults to 1 million hashes.

Returns the time taken in milliseconds and speed in megahashes/second.

E.g. A speed of 0.5 MH/s indicates 500000 hashes per second.

ParametersDescription
amount
(Optional) Number of hashes to execute.

Examples:

History

Return all TxPoW relevant to you.

ParametersDescription
max
(Optional) Maximum number of TxPoW to retrieve.
offset
(Optional) Start the list from this point. (v1.0.41 onwards)

Examples:

Keys

Get a list of all your public keys or create a new key.

ParametersDescription
action
(Optional) action:list, action:checkkeys, action:new, action:genkey.
publickey
(Optional) Search for a specific public key.
phrase
(Optional) The phrase to use to generate the private/public key pair.

Parameters used with the action: parameter:

ParametersDescription
checkkeys
action:checkkeys Checks if your Public and Private keys are correct.
genkey
action:genkey Generate a new private/public key pair and wallet address from a new random seed phrase or from a given phrase. This is independent from the node's wallet.
list
action:list List your existing public keys. The default.
new
action:new Create a new key pair.

Examples:

Newaddress

Create a new address that will not be not used for anything else (not one of the 64 default change address).

Can be used for a specific use case or for improved privacy.

Printmmr

Print the MMR set of the tip block and the total number of entries in the MMR.

Returns the tip block number, latest entrynumber and latest set of MMR entries.

For each entry, details of its row, entry number, data and value of all new and updated MMR entries for the tip block.

Row 1 represents the leaf nodes, entry 0 represents the first entry on a row.

Printtree

Print a tree representation of the blockchain.

Default depth 32 blocks, can be increased to see more of the txpow tree.

ParametersDescription
depth
(Optional) Number of blocks back from the tip to show in the txpow tree.
cascade
(Optional) true or false, true shows the cascade.

Examples:

Quit

Shutdown Minima safely. Ensure you have a backup before shutting down.

Ensure you have a backup before shutting down.

Examples:

Seedrandom

Generate a random value, based on your SEED and a modifier.

ParametersDescription
modifier
The modifier - added to seed before hash.

Examples:

Status

Show the general status for Minima and your node. Optionally clean the RAM.

Prints details for general status, memory used, chain info, stored txpow units, network connections, p2p connections and traffic.

ParametersDescription
clean
(Optional) true only, clear the RAM.

Examples:

Tokencreate

Create (mint) custom tokens or NFTs.

You must have some sendable Minima in your wallet as tokens are 'colored coins', a fraction of 1 Minima.

ParametersDescription
name
The name of the token. Can be a string or JSON Object.
amount
The amount of total supply to create for the token. Between 1 and 1 Trillion.
decimals
(Optional) The number of decimal places for the token. Default is 8, maximum 16.
script
(Optional) Add a custom script that must return 'TRUE' when spending any coin of this token.
state
(Optional) List of state variables, if adding a script.
signtoken
(Optional) Provide a public key to sign the token with. Useful for proving you are the creator of the token/NFT.
webvalidate
(Optional) Provide a URL to a publicly viewable .txt file you are hosting which stores the tokenid for validation purposes.
burn
(Optional) Amount to burn with the tokencreate minting transaction.

Examples:

Tokenvalidate

Validate the signature and webvalidate link in a token.

ParametersDescription
tokenid
The tokenid of the custom token/NFT to validate.

Examples:

Trace

Show the message stacks of the internal Minima Engine with optional filter string.

ParametersDescription
enable
true or false, true to enable or false to disable.
filter
(Optional) A case-sensitive string to filter the messages by.
